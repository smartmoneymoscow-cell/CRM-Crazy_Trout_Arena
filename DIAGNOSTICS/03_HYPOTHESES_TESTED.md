# Проверенные гипотезы

## Гипотеза 1: ProGuard/R8 вырезает классы Flutter
ПРОВЕРЕНА: ❌ НЕ ПОДТВЕРДЖЕНА
- minifyEnabled=false по умолчанию (flutter 3.44)
- R8 не запускается → ProGuard правила не применяются
- v1.3.3/v1.3.5 работали тоже без minification

## Гипотеза 2: Конфликт workflow (auto-release.yml + build-apk.yml)
ПРОВЕРЕНА: ✅ ЧАСТИЧНО ПОДТВЕРЖЕНА
- Оба workflow стреляли на push tag v*
- APK в релизе мог быть из другого workflow run
- v1.3.5 APK был собран из старого коммита (не v1.3.4)
- auto-release.yml удалён, но проблема крашей осталась

## Гипотеза 3: Sentry вызывает краш
ПРОВЕРЕНА: ❌ НЕ ПОДТВЕРДЖЕНА
- sentry_flutter 8.x: Kotlin 1.6 vs Flutter 3.44 Kotlin 2.0 → build failure
- sentry_flutter 9.x: build прошёл, но краш остался
- Sentry полностью убран → краш остался

## Гипотеза 4: Fish images вызывают краш
ПРОВЕРЕНА: ❌ НЕ ПОДТВЕРДЖЕНА
- PNG файлы валидные (200x100)
- Код не загружает fish images при запуске (только при добавлении рыбы)
- Fish images убраны из pubspec → краш остался (v1.3.9)

## Гипотеза 5: Проблема в изменениях v1.3.4
ПРОВЕРЕНА: ⏳ В ПРОЦЕССЕ
- v1.3.11 = код v1.3.3 + текущий workflow — СОБИРАЕТСЯ СЕЙЧАС
- Если v1.3.11 работает → проблема в коде v1.3.4
- Если v1.3.11 крашится → проблема в workflow

## Гипотеза 6: Проблема в QR scanner (убрана задержка 500ms)
ПРОВЕРЕНА: ❌ МАЛОВЕРОЯТНА
- QR scanner загружается через deferred
- Не инициализируется при запуске
- Только при нажатии кнопки сканера

## Гипотеза 7: Проблема в flutter create в CI
ПРОВЕРЕНА: ⏳ ЧАСТИЧНО
- flutter 3.44 генерирует build.gradle.kts с isMinifyEnabled=false
- Python-скрипт в workflow НЕ меняет его (находит ключевое слово, считает что настроено)
- Минификация отключена → не может быть причиной краша
- НО: flutter create может генерировать что-то отличное от локальной сборки
