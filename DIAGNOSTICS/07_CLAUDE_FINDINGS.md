# Находки со стороны (параллельная диагностика без доступа к устройству)

Проверено методом реверс-инжиниринга опубликованных APK напрямую (без
Flutter SDK и без Android-устройства): скачивание готового `.apk` через
GitHub Releases API, распаковка как zip, разбор `AndroidManifest.xml` /
`resources.arsc` / `classes.dex` через `androguard`. Инструмент воспроизводим
локально, если понадобится (`pip install androguard`).

## 3. Sentry подключён в коде НЕПОЛНОЦЕННО — вероятная причина, почему
## "Sentry не причина" в 03_HYPOTHESES_TESTED.md может быть ложным выводом

В `0b5cbd4`/`792f8a7` DSN передавался ТОЛЬКО через `SentryFlutter.init()` в
Dart-коде (`main.dart`), который выполняется ПОСЛЕ старта Flutter-движка.
Если краш происходит на нативном уровне ДО того, как Dart-код успевает
запуститься (что вполне вероятно, учитывая симптом "мгновенный вылет на
домашний экран без диалога" — характерно для native SIGSEGV/JNI-краша, а
не для пойманного Dart-исключения) — Sentry просто не успевает
инициализироваться и ничего не отправляет.

**Также не проверено: смотрели ли вообще в сам дашборд Sentry после сборки
v1.3.7/v1.3.8?** В `03_HYPOTHESES_TESTED.md` написано только "краш остался"
(со стороны телефона), но не указано, было ли событие в Sentry UI. DSN в
коммите `792f8a7` — настоящий, не заглушка. Стоит проверить дашборд ДО
пересборки чего-либо — это бесплатно и может дать готовый crash trace уже
сейчас.

## 4. Рекомендация: native-level DSN, не только Dart-level

Чтобы поймать краши ДО инициализации Dart/Flutter, DSN нужно продублировать
в `android/app/src/main/AndroidManifest.xml` как meta-data — тогда нативный
Android-слой Sentry SDK стартует в момент запуска процесса приложения,
независимо от того, успел ли вообще выполниться Dart-код:

```xml
<meta-data android:name="io.sentry.dsn" android:value="<DSN>" />
<meta-data android:name="io.sentry.anr.enable" android:value="true" />
```

Собираюсь запушить патч с этим (см. следующий коммит) поверх текущего
рабочего build-apk.yml (без изменения кода экранов — чтобы не смешивать
переменные).
